Phi Ï† Code Documentary


Phi Code's idea is based on a specific language, and inspired the birth of Phi Code.
It is tested that it is possible to solve all puzzle by only using the following codes, so you do not have to worry about concatting strings.

1.Output

To output some context, use ~ to represent printing. use space to separate each character, after ~ and before ;. Also use ascii. In this example of printing out Hello World,
~ 72 101 108 108 111 32 87 111 114 108 100 ;

Expected output:
Hello World

2.Ending Syntax

Every Phi Code statement ends with a semicolon ;

3.Variable

There does not exist variable in Phi Code. But instead, you can allocate memory from the array. Use @ to denote you are referring to the memory bar. For example storing the number 10 in the first address of the array,
@0 : 10 ;
 To output the first address,
~ @0 ;
 Expected Output:
10

4.User Input

Use > to get user input. Be reminded that for length >= 2 input, the system will automatically replace the following address memory for enough space. For example getting user input and storing it in the 2nd address,
> @1 ; ~ @1 ;
 User Input:
a
 Expected Output:
a
> @2 ; @5 : f ; ~ @5 ;
 User Input:
IS Club
 Expected Output:
C

5.Arithmtic Operation

There does not exist + - * /. Instead, & | ^ are used.
https://en.wikipedia.org/wiki/Bitwise_operation
For example, finding the modulo of user input by 4,
> @0 ; ~ @0 & 3 ;
 User Input:
89
 Expected Output:
1

6.Logic Operation and If-Else Condition

Use = > < to compare between numbers. Phi Code uses tenary operator for If-Else condition. However for each ternary operator, you have to use increasing ? : For example, determining is the user input non-positive, then determining is it zero or not
> @0 ; ~ @0 < 1 ? @0 > -1 ?? 90 101 114 111 :: 78 101 103 97 116 105 118 101 : 80 111 115 105 116 105 118 101 ;
 User Input: 10
Expected Output:
Positive
